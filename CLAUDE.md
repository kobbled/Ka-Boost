# Ka-Boost — CLAUDE.md

## Project Overview

**Ka-Boost** is a collection of FANUC Karel library packages managed as git submodules. Each package is formatted for the [rossum](https://github.com/kobbled/rossum) Karel package manager. The overarching goal is a **5-axis DLP 3D printing slicer** that drives a FANUC robot arm, converting 2D slice geometry from UV/parameterized space into full `XYZWPR` robot motion paths.

Karel is a Pascal-like compiled language that runs natively on FANUC robot controllers. It lacks generics, associative arrays, native string manipulation, or a standard library — Ka-Boost exists to fill those gaps.

---

## Toolchain

| Tool | Role |
|------|------|
| [rossum](https://github.com/kobbled/rossum) | Karel package manager — resolves submodule dependencies, generates `build.ninja` |
| [ktransw](https://github.com/kobbled/ktransw_py) | Karel transpiler/preprocessor (wraps GPP) |
| [GPP](https://github.com/logological/gpp) | C-style preprocessor used for generics (`.klc` class files, `.m` macro files) |
| [TP-Plus](https://github.com/kobbled/tp_plus) | High-level DSL that compiles to Fanuc TP (`.ls`) programs |
| [yamljson2xml](https://github.com/kobbled/yamljson2xml) | Bundled with rossum; converts config to XML |
| ninja | Build system (generated by rossum) |
| kpush | Deploys compiled `.pc` files to the robot controller over FTP |

### File Extensions

| Extension | Meaning |
|-----------|---------|
| `.kl` | Karel source file |
| `.klc` | Karel class (template, expanded by GPP) |
| `.klt` | Karel type/constant definitions |
| `.klh` | Karel header (function declarations) |
| `.m` | GPP macro definition file |
| `.tpp` | TP-Plus source file |
| `.ls` | Fanuc TP source (ASCII) |
| `.pc` | Compiled Karel binary (loaded onto controller) |

### Build Flow
```
rossum .. -w -o     # generate build.ninja from package.json dependency graph
ninja               # compile all .kl/.klc files → .pc binaries
kpush               # FTP-push .pc files to the robot controller
```

## Repository Structure

```
Ka-Boost/
├── lib/                 # All library submodules (each is a separate git repo)
├── docs/                # Generated HTML documentation
├── scripts/             # Windows batch scripts (master_del.bat, master_test_del.bat)
├── .claude/rules/       # Per-layer module detail docs
├── Ka-Boost.code-workspace
└── readme.md
```

Each submodule in `lib/` contains a `package.json` (rossum manifest) with `name`, `version`, `depends` (other Ka-Boost packages), and optional `tp-interfaces` (routines exposed to TP programs).

## Key Architectural Patterns

### Generics via GPP
Karel has no generics. Ka-Boost emulates them with `.klt` config files + `%class` macros + GPP:
```
%define class_name myqueue
%include default_queue.klt     -- defines QUEUE_INDEX_TYPE, QUEUETYPE, etc.
%class myqueue(queue.klc)      -- expand the class template
```

### Namespace Scoping
All public routines follow `modulename__routine` naming (double underscore), generated by `funcname`/`classfunc` macros from `ktransw-macros`. Short aliases are registered via `declare_function`.

### OOP Classes (`.klc`)
Object state is stored in Karel PATH structures or global variables. Methods use the `declare_member` macro to bind to a class instance. Classes are instantiated by including a config `.klt` and expanding via `%class`.

### Testing
All modules include tests using `lib/KUnit`. Tests run on the robot controller and are viewable via HTTP. Config test files live in `tests/config/`.

## Building a Specific Module

```shell
cd lib/<module_name>
del /f robot.ini && setrobot   # configure target robot/roboguide instance
mkdir build && cd build
rossum .. -w -o                # resolve deps + write build.ninja
ninja                          # compile
kpush                          # deploy to controller
```

To include test programs: `rossum .. -w -o -t`
To build all dependencies: `rossum .. -w -o -b`

## Controller Cleanup

```shell
kpush --delete                          # remove all programs in build dir from controller
cd scripts && ./master_del.bat          # remove all Ka-Boost programs
cd scripts && ./master_test_del.bat     # remove all test programs
```

> **Warning:** Cross-dependencies between `draw` and `paths` can cause `MEMO-128 parameters are different` errors. Run `master_del.bat` + `master_test_del.bat` before re-deploying.
